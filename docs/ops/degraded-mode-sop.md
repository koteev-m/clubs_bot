# Degraded mode SOP: Mini App недоступен → fallback в бот

## Триггеры перехода в degraded mode

- Mini App не открывается/отдаёт 5xx.
- Ошибки валидации initData блокируют вход.
- Отказ внешних зависимостей, влияющих на web-часть.

## Цель

Сохранить критичный путь гостя и персонала через Telegram-бот:
- статус брони;
- check-in/поиск гостя;
- базовые уведомления и поддержка.

## Операционный порядок

1. Подтвердить инцидент (метрики + алерты + `/ready`).
2. Объявить degraded mode в ops-канале.
3. Включить fallback-маршруты/команды в боте (если есть feature-flag — включить).
4. Убрать/скрыть CTA Mini App в коммуникациях и дать пользователям ссылку на bot-flow.
5. Хостам выдать краткую инструкцию: поиск гостя и check-in через бот/журнал.
6. Мониторить очередь инцидентов и SLA поддержки.

## Критерии выхода из degraded mode

- Mini App стабильно проходит `/ready` и smoke.
- 15+ минут без новых ошибок уровня P1/P2.
- Подтверждён успешный пользовательский сценарий end-to-end.

## После инцидента

- Пост-инцидентный разбор.
- Обновление алертов и fallback-процедур.
